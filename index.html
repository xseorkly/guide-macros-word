Guide de démarrage
1

Ouvrir l'éditeur VBA

Appuyez sur Alt + F11 dans Word. L'éditeur Visual Basic s'ouvre dans une fenêtre séparée.

2

Créer un module

Dans le menu de l'éditeur, cliquez sur Insertion → Module. Un module vide apparaît sous le document ou le Normal.dotm dans l'explorateur de projet.

3

Coller le code VBA

Cliquez dans la zone blanche du module et collez le code. Chaque macro commence par Sub NomMacro() et se termine par End Sub.

4

Tester la macro

Cliquez à l'intérieur du code, puis appuyez sur F5 pour exécuter. Utilisez F8 pour exécuter ligne par ligne (débogage pas à pas).

5

Enregistrer dans Normal.dotm

Pour qu'une macro soit disponible dans tous vos documents, placez-la dans le projet Normal (Normal.dotm) plutôt que dans le document en cours.

6

Associer à un bouton ou raccourci

Onglet Développeur → Macros → sélectionner la macro → Options pour lui assigner un raccourci clavier. Ou : Fichier → Options → Barre d'outils Accès rapide.

🔧 Activer l'onglet Développeur

Fichier → Options → Personnaliser le ruban → cocher Développeur. Donne accès à l'éditeur VBA, aux macros et aux contrôles de contenu.

🔐 Niveau de sécurité des macros

Développeur → Sécurité des macros → "Désactiver toutes les macros avec notification". Word demandera confirmation à chaque ouverture d'un fichier avec macros.

📂 Emplacement approuvé

Fichier → Options → Centre de gestion de la confidentialité → Emplacements approuvés. Les fichiers de ce dossier s'ouvrent avec macros activées automatiquement.

📚 Références VBA Word

Dans l'éditeur VBA : Outils → Références. Pour Excel, cochez Microsoft Excel Object Library. Pour Outlook, Microsoft Outlook Object Library.

💾 Normal.dotm — modèle global

Les macros dans Normal.dotm sont disponibles dans tous vos documents. Celles dans un .docm sont limitées à ce fichier. Placez vos macros utilitaires dans Normal.dotm.

📝 Format .docm obligatoire

Pour conserver les macros dans un document, enregistrez en .docm (Document Word avec macros activées). Le format .docx supprime silencieusement tout le code VBA.

⛔

Oublier de sauvegarder en .docm

Comme pour Excel, le format .docx ne supporte pas les macros. Tout votre code VBA sera perdu à la prochaine sauvegarde si vous ne choisissez pas .docm.

Toujours choisir "Document Word prenant en charge les macros (*.docm)" lors de l'enregistrement.
⚠️

Utiliser Selection plutôt que Range

Selection dépend de ce que l'utilisateur a sélectionné manuellement. Si rien n'est sélectionné ou si la sélection change, la macro plante ou donne des résultats imprévisibles.

Toujours utiliser des objets Range définis par le code : ActiveDocument.Range(Start:=0, End:=100).
⛔

Ne pas gérer les erreurs

Si un signet est absent, un fichier introuvable ou un style inexistant, la macro plante sans message clair. En Word, les signets manquants sont la cause d'erreur n°1.

Toujours vérifier l'existence avant d'utiliser : If ActiveDocument.Bookmarks.Exists("MonSignet") Then.
⚠️

Modifier le document sans sauvegarder l'original

Une macro qui remplace, supprime ou reformate du contenu est irréversible. Ctrl+Z peut fonctionner partiellement mais pas toujours sur les modifications VBA complexes.

Créer une copie avant exécution : ActiveDocument.SaveAs2 chemin & "_backup.docx" ou demander confirmation avec MsgBox.
⛔

Ne pas désactiver le suivi des modifications

Si le suivi des modifications est actif, chaque action VBA génère une révision dans le document. La macro peut fonctionner mais le document devient illisible avec des milliers de marques.

Ajouter en début de macro : ActiveDocument.TrackRevisions = False et le réactiver à la fin si nécessaire.
⚠️

Utiliser des noms de styles codés en dur

Les styles Word ont des noms différents selon la langue de Word ("Titre 1" en français, "Heading 1" en anglais). Une macro qui fonctionne sur votre poste peut échouer chez un collègue.

Utiliser les ID de style intégrés : wdStyleHeading1 au lieu de "Heading 1" pour les styles standards.
ℹ️

Négliger la boucle Find/Replace

La méthode Find de Word nécessite toujours un .ClearFormatting avant chaque utilisation. Sans cela, les critères de recherche précédents s'accumulent et donnent des résultats inattendus.

Toujours appeler .Find.ClearFormatting et .Replacement.ClearFormatting avant chaque opération de recherche.
